# Human-Facing Stage Reports

This directory stores human-facing stage reports produced by Codex.

Naming convention:
- `stage_0.md`
- `stage_1.md`
- `stage_2.md`
- ...

Rules:
- Each stage report must be written before moving to the next stage.
- Reports should summarize what happened, what evidence exists, what risks remain, and what decision is recommended.
- These files are for project-level supervision and should be concise, factual, and auditable.
- Do not delete or overwrite prior stage reports unless explicitly instructed by the human supervisor.
